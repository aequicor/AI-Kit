# M4 — Экспорт (Flatten в PDF)

**Created:** 2026-05-12
**Branch:** feature/m4-export
**Source task:** M4 — «вжечь» аннотации в PDF (flatten) и сохранить копию.
PDFBox на JVM, PDFKit на iOS, iText/custom на Android.

## Context (digest)

- Строится поверх M3: все типы инструментов готовы, Stroke хранится в JSON.
- Flatten = перерисовать каждый Stroke на PDF-странице через платформенный PDF-движок.
- Stroke нормализованы в PDF-координаты → прямое маппирование в PDF user space.
- Android: нет встроенного API для записи PDF с рисованием (PdfRenderer — read-only).
  Вариант: iText7 Community (AGPL, риск лицензии) или собственный рендеринг через
  `android.graphics.Canvas` + `android.print.PrintedPdfDocument` (API 19+).
  Предпочитаем `PrintedPdfDocument` / `PdfDocument` (android.graphics) — без сторонних зависимостей.
- Desktop: PDFBox `PDPageContentStream` + `PDFormXObject` для рисования поверх страницы.
- iOS: `PDFDocument` + `PDFAnnotation` или UIGraphicsBeginPDFContextToData (Assumption: macOS).
- Export — тяжёлая операция, всегда на `Dispatchers.IO` с прогресс-колбэком.

## Invariants

- Исходный PDF-файл не изменяется; экспорт создаёт новый файл рядом с оригиналом.
- `ExportPort` interface — в `shared/domain/port/`, реализации — в platform source-sets.
- Export-операция отменяема через `CoroutineScope` cancellation.
- Новые public API в `shared` покрыты unit-тестами.
- `server/` не изменяется.

## Steps

### Step 1 — export-domain
- **Goal:** `ExportPort` interface: `suspend fun exportToPdf(documentId, outputPath, progress: (Float)->Unit): ExportResult`;
  `ExportResult` sealed class (Success(path), Failure(error));
  `ExportDocumentUseCase`; unit-тест с fake port.
- **DoD:** `./gradlew.bat :shared:jvmTest` зелёный.
- **Review:** standard
- **What would be wrong:** `ExportPort` принимает `File` вместо `String` path → java.io.File
  утекает в shared/commonMain — нарушение инварианта.
- **Verify:** `shell: "./gradlew.bat :shared:jvmTest"`
- **Expect:** green

### Step 2 — desktop-export
- **Goal:** `JvmExportAdapter : ExportPort` в `composeApp/jvmMain` через PDFBox:
  `PDDocument.load(source)` → для каждой страницы `PDPageContentStream` → рисуем Stroke
  как `lineTo`/`curveTo` в PDF user space → `PDDocument.save(output)`;
  прогресс по страницам; save-picker через JFileChooser.
- **DoD:** `./gradlew.bat :composeApp:compileKotlinJvm` exit 0.
- **Review:** standard
- **What would be wrong:** `PDPageContentStream` открывается в режиме APPEND без явного указания
  → содержимое страницы замещается вместо наложения аннотаций.
- **Verify:** `shell: "./gradlew.bat :composeApp:compileKotlinJvm"`
- **Expect:** green

### Step 3 — android-export
- **Goal:** `AndroidExportAdapter : ExportPort` в `composeApp/androidMain`:
  читаем PDF через `PdfRenderer`, для каждой страницы — `Bitmap` + рисуем Stroke через `Canvas`,
  собираем результирующий PDF через `android.graphics.pdf.PdfDocument` (API 19+);
  save-picker через `ActivityResultContracts.CreateDocument("application/pdf")`.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0.
- **Review:** standard
- **What would be wrong:** `android.graphics.pdf.PdfDocument` пишет страницы как растр (не вектор)
  — PDF будет большим; для документа 100 страниц может не хватить памяти без постраничного flush.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug"`
- **Expect:** green
- **Assumptions:** результирующий PDF на Android растровый (приемлемо для M4, векторный — M8).

### Step 4 — ios-export
- **Goal:** `IosExportAdapter : ExportPort` в `composeApp/iosMain` через PDFKit:
  `PDFDocument` → `PDFPage` → `UIGraphicsBeginPDFContextToData` + рисование через CoreGraphics;
  UIDocumentPickerViewController для сохранения.
- **DoD:** Assumption: requires macOS.
  `./gradlew :composeApp:linkDebugFrameworkIosSimulatorArm64` exit 0.
- **Review:** standard
- **What would be wrong:** CoreGraphics CTM (coordinate transform matrix) у PDF-страниц
  перевёрнутый по Y относительно UIKit — штрихи окажутся зеркальными по вертикали.
- **Verify:** `shell: "./gradlew :composeApp:linkDebugFrameworkIosSimulatorArm64"`
- **Expect:** green
- **Assumptions:** macOS host required.

### Step 5 — export-ui
- **Goal:** кнопка «Экспорт» в toolbar; `ExportProgressDialog` (LinearProgressIndicator по
  страницам); toast/snackbar по завершении с путём файла; отмена через кнопку Cancel
  (cancels coroutine); `ExportViewModel` с `StateFlow<ExportState>`.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm` exit 0.
- **Review:** standard
- **What would be wrong:** `(n/a)` — стандартный UI, основной риск в адаптерах (steps 2–4).
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm"`
- **Expect:** green

## Out of scope

- Векторный PDF на Android (M8).
- Экспорт в PNG/JPEG (не требуется).
- Экспорт только выбранных страниц (M8).
- Шифрование результирующего PDF (не требуется).
