## Goal
Довести приложение до releasable состояния: профилировать и оптимизировать рендеринг и input latency, обеспечить accessibility, исправить накопленные баги, настроить релизные сборки для всех платформ.

## Steps

### Step 1 — Performance audit: рендеринг PDF
**Commit:** `perf: PDF page render — bitmap cache tuning + background prefetch`

- Профилирование на Android (Android Studio Profiler), Desktop (JVM Flight Recorder), iPad (Instruments)
- Целевые метрики: первый рендер страницы <300 мс, scroll jank <1 дроп/сек на 60fps
- Оптимизации:
  - Увеличить LRU кэш до `min(10, RAM / 20MB)` страниц
  - Prefetch: при скролле вниз — рендерить +2 страницы вперёд в `Dispatchers.Default`
  - Android: `Bitmap.Config.RGB_565` вместо `ARGB_8888` для страниц без прозрачности (-50% RAM)
  - Desktop: `PDFRenderer.renderImageWithDPI` кэшировать при неизменном scale

---

### Step 2 — Performance audit: input latency
**Commit:** `perf: stylus input latency — predictive rendering + frame pacing`

- Измерение: логировать `StylusEvent.timestamp` vs `drawFrame` timestamp через `Kermit`
- Оптимизации:
  - Predictive rendering: последний сегмент штриха в dedicated `Canvas` layer без invalidate всего экрана
  - Android: `requestUnbufferedDispatch()` для `MotionEvent` (снижает буферизацию OS)
  - Compose: `Modifier.graphicsLayer` для overlay-слоя (GPU compositing без recomposition)
  - Desktop: проверить `ComposePanel` vs `ComposeWindow` frame pacing

---

### Step 3 — Accessibility
**Commit:** `feat(composeApp): accessibility — semantics, contrast, tap targets`

- `Modifier.semantics { contentDescription = "PDF page $index of $total" }` для каждой страницы
- `Modifier.semantics { role = Role.Button }` для тулбарных кнопок с `contentDescription`
- Минимальный tap target: `Modifier.sizeIn(minWidth = 44.dp, minHeight = 44.dp)`
- Цветовая палитра: проверить контраст через `Color.luminance()`, все цвета ≥ 4.5:1 на белом фоне
- Android: TalkBack совместимость — тест `accessibilityNodeInfo`
- iOS: VoiceOver labels через `Modifier.semantics`
- Системные шрифты: использовать `MaterialTheme.typography` везде, не хардкодить `fontSize`

---

### Step 4 — Error handling & edge cases
**Commit:** `fix: error handling — corrupt PDF, network errors, disk full`

- Corrupt/password-protected PDF: `PdfRenderer.openDocument` возвращает `Result`, UI показывает `AlertDialog`
- Network disconnect во время transfer: retry с exponential backoff, UI показывает статус
- Disk full при экспорте: проверять свободное место перед записью, `Result.Failure` с понятным сообщением
- Large PDF (>200 страниц): тест на Pixel 6, убедиться что LazyColumn не OOM
