# M8 — Полировка

**Created:** 2026-05-12
**Branch:** feature/m8-polish
**Source task:** M8 — производительность, accessibility, баг-фикс, релизные сборки.

## Context (digest)

- Все функциональные milestones M1–M7 завершены. M8 — финальная итерация качества.
- Профилирование: Android Profiler, Instruments (macOS), JVM Flight Recorder для Desktop.
- Ключевые метрики: scroll/zoom 60fps, input latency <16ms (touch → stroke appear), projection <100ms.
- Accessibility: WCAG AA contrast, tap-targets ≥44dp, contentDescription для иконок, TalkBack/VoiceOver.
- Release: Android — ProGuard/R8 + upload-ready AAB; Desktop — native package (Dmg/Msi/Deb через compose.desktop); iOS — Archive via Xcode (Assumption: macOS).
- Опциональные улучшения: PencilKit integration для iOS (если Compose MP PointerType.Stylus недостаточно), векторный PDF export на Android, tile-based page cache.

## Invariants

- Никакие публичные API не изменяются без согласования (M8 — только внутренние оптимизации).
- Все unit-тесты из M1–M7 остаются зелёными.
- Release-сборки не содержат debug-артефактов (логи Kermit на уровне WARN, нет LeakCanary).
- `minSdk = 26`, `targetSdk = 36` зафиксированы.

## Steps

### Step 1 — perf-page-rendering
- **Goal:** tile-based page cache: делить страницу на 4 тайла, рендерить только видимые тайлы;
  `RenderedPageCache` (LRU, ограничен 200 МБ); pre-render следующей и предыдущей страниц
  на `Dispatchers.Default`; отменять рендеринг при быстрой прокрутке через `Job.cancel()`.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0; измерить frame time на Android с
  50-страничным PDF — цель <16ms avg.
- **Review:** standard
- **What would be wrong:** pre-render не отменяется при быстрой прокрутке → очередь задач
  разрастается, рендеринг отстаёт на несколько секунд.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm"`
- **Expect:** green

### Step 2 — perf-stroke-rendering
- **Goal:** инкрементальный рендеринг штрихов: хранить pre-rasterized bitmap завершённых штрихов
  (`drawIntoCanvas` → offscreen `ImageBitmap`), добавлять только новый in-progress штрих поверх;
  invalidate bitmap только при undo/redo/delete; Catmull-Rom вычисляется инкрементально (только
  новые сегменты).
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0.
- **Review:** standard
- **What would be wrong:** offscreen bitmap перерисовывается при каждом recomposition вместо
  только при изменении Stroke-коллекции → съедает всё преимущество кэширования.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm"`
- **Expect:** green

### Step 3 — accessibility
- **Goal:** добавить `contentDescription` ко всем иконкам toolbar; проверить контрастность
  цветовой палитры (WCAG AA — 4.5:1 для обычного текста, 3:1 для UI-элементов);
  tap-targets ≥44dp везде; добавить `semantics { role = Role.Button }` где нужно;
  поддержка системного шрифта (не хардкодить `fontFamily`); проверить TalkBack-навигацию.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0; ручная проверка TalkBack на Android.
- **Review:** standard
- **What would be wrong:** `(n/a)` — преимущественно декларативные изменения.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm"`
- **Expect:** green
- **Shape:**
    - **files-glob:** `**/composeApp/src/**/*.kt`
    - **max-diff-lines:** 200
    - **no-test-changes:** false

### Step 4 — release-config-android
- **Goal:** включить R8/ProGuard в release buildType; добавить rules для Koin, Ktor, kotlinx.serialization,
  PDFBox (если используется через reflection); подписать AAB через keystore (placeholder config,
  реальный keystore — вне репозитория); проверить что Kermit на уровне WARN в release.
- **DoD:** `./gradlew.bat :composeApp:bundleRelease` exit 0.
- **Review:** heavy
- **What would be wrong:** ProGuard удаляет `@Serializable` generated classes → `SerializationException`
  в release-сборке; нужен keep rule `**/serializer/**`, `**$$serializer*`.
- **Verify:** `shell: "./gradlew.bat :composeApp:bundleRelease"`
- **Expect:** green
- **Shape:**
    - **files-glob:** `**/{build.gradle.kts,*.pro,proguard-rules.pro,libs.versions.toml}`
    - **max-diff-lines:** 100
    - **no-test-changes:** true

### Step 5 — release-config-desktop
- **Goal:** `compose.desktop.application` — настроить `nativeDistributions` (packageName,
  packageVersion, icons, copyright, vendor); `./gradlew :composeApp:packageDmg` (macOS) /
  `packageMsi` (Windows) / `packageDeb` (Linux); убедиться что JDK bundled, не system.
- **DoD:** `./gradlew.bat :composeApp:packageMsi` exit 0 (Windows-хост).
- **Review:** standard
- **What would be wrong:** `includeAllModules = true` в JDK bundling тянет весь JDK (~300MB)
  вместо только нужных модулей; использовать `modules = listOf(...)` с минимальным набором.
- **Verify:** `shell: "./gradlew.bat :composeApp:packageMsi"`
- **Expect:** green
