# M3 — Палитра инструментов

**Created:** 2026-05-12
**Branch:** feature/m3-toolbar
**Source task:** M3 — маркер, ластик (объектный + пиксельный), выбор цвета, выбор толщины,
выбор страницы, миниатюры страниц.

## Context (digest)

- Строится поверх M2: Stroke domain готов, overlay canvas работает, undo/redo есть.
- Маркер = кисть с `blendMode = BlendMode.Multiply` или фиксированной прозрачностью alpha ~0.4.
- Ластик объектный: hit-test штриха (AABB + path proximity); ластик пиксельный: mask/erase layer.
- Миниатюры: рендеринг каждой страницы в низком разрешении (1/8 от оригинала), кэш в памяти + диск.
- Jump-to-page через LazyListState.scrollToItem.
- ToolSettings как часть UI-state (не domain) — не персистируются между сессиями в M3.

## Invariants

- Tool-switching не создаёт новый Stroke до первого touch event — инструмент выбран, штрих не начат.
- Ластик объектный не мутирует существующие Stroke — удаляет их целиком через `DeleteStrokeCommand`.
- Миниатюры рендерятся на `Dispatchers.Default`, не на Main thread.
- Новые public API в `shared` покрыты unit-тестами.
- `server/` не изменяется.

- **Goal:** `ToolSettings` data class (activeTool, color: Color ARGB Long, strokeWidth: Float);
  `ToolSettingsRepository` port (in-memory в M3); `SelectToolUseCase`, `SetColorUseCase`,
  `SetWidthUseCase`; unit-тесты.
- **DoD:** `./gradlew.bat :shared:jvmTest` зелёный.
- **Review:** standard
- **What would be wrong:** `Color` из `androidx.compose.ui.graphics` попадает в `shared/commonMain`
  — нарушение инварианта; вместо этого хранить как `Long` (ARGB).
- **Verify:** `shell: "./gradlew.bat :shared:jvmTest"`
- **Expect:** green

### Step 2 — toolbar-ui

  Thickness slider (1dp–20dp), Undo/Redo; привязка к `ToolSettingsViewModel`; минимальный

- **Review:** standard
- **What would be wrong:** toolbar перекрывает PDF-контент без возможности свернуть;
  Color picker открывается как отдельное Activity вместо bottom sheet.

- **Expect:** green

### Step 3 — marker-tool
- **Goal:** `MarkerStrokeRenderer` — рендеринг с `Paint.alpha = 102` (~40%) и
  `BlendMode.Multiply`; единый path для всего штриха маркера (не инкрементальный как у кисти,
  чтобы перекрытия не накапливались); обновление StrokeOverlayCanvas при смене tool.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0.
- **Review:** standard
- **What would be wrong:** incremental rendering маркера накапливает alpha в точках перекрытия
  внутри одного штриха — маркер выглядит как непрозрачная кисть.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug"`
- **Expect:** green

### Step 4 — object-eraser
- **Goal:** `ObjectEraserHandler` — на каждый PointerMove вычислять расстояние от точки до каждого
  Stroke (AABB pre-filter, затем Bézier path distance); при попадании — `DeleteStrokeCommand`;
  visual feedback — stroke мигает/исчезает немедленно; unit-тест для hit-test логики.
- **DoD:** `./gradlew.bat :shared:jvmTest :composeApp:assembleDebug` зелёный.
- **Review:** standard
- **What would be wrong:** O(N²) hit-test по всем штрихам на каждый event — лаг при >100 штрихах;
  нужен spatial index (простой grid или AABB tree).
- **Verify:** `shell: "./gradlew.bat :shared:jvmTest :composeApp:assembleDebug"`
- **Expect:** green

### Step 5 — pixel-eraser
- **Goal:** `PixelEraserHandler` — рендеринг маски `BlendMode.Clear` поверх stroke layer;
  ластик работает с растеризованным слоем аннотаций (offscreen Canvas c `saveLayer`);
  хранить pixel-mask как `List<EraseRect>` в Stroke специального типа `PIXEL_ERASE`; undo.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug` exit 0.
- **Review:** standard
- **What would be wrong:** `BlendMode.Clear` работает только внутри `saveLayer` — без него
  очищает весь экран; отдельный `saveLayer` на каждый frame → perf cliff.
- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug"`
- **Expect:** green

### Step 6 — page-thumbnails
- **Goal:** `ThumbnailCache` (LRU, max 50 entries, 1/8 разрешения) в `shared/data/`;

  LazyRow внизу экрана или в side drawer; рендеринг thumbnail на `Dispatchers.Default`.
- **DoD:** `./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm` exit 0.
- **Review:** standard
- **What would be wrong:** ThumbnailCache хранит `ImageBitmap` (GPU texture) без чёткого

- **Verify:** `shell: "./gradlew.bat :composeApp:assembleDebug :composeApp:compileKotlinJvm"`
- **Expect:** green
